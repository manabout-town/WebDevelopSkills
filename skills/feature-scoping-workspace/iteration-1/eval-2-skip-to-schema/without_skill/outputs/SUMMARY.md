# 요약

provider 평점/리뷰 기능에 대해 두 가지 스키마 안을 제시했습니다: (1) 평점과 후기를 하나의 `reviews` 테이블에 합치는 안(추천), (2) 사용자가 요청한 대로 `ratings`와 `reviews`를 분리하는 안. 각 컬럼의 역할과 제약조건(중복 리뷰 방지, 평균 평점 캐싱 등)을 설명하고, 분리 여부를 결정하기 위해 거래 단위 존재 여부·다항목 평점 필요성·리뷰 수정 허용 여부 등 5가지 확인 질문을 던지며 마무리했습니다. 별도의 요구사항 정의나 우선순위 논의 없이 곧바로 SQL 스키마 설계로 들어간 응답입니다.
