# Summary

feature-scoping 스킬을 적용해 "게시물 북마크" 기능에 대한 스코프 문서(response.md)를 작성했다. 문제 정의, 단일 액터(로그인 사용자) 명시, 번호가 매겨진 사용자 플로우(비로그인 분기·빈 상태·삭제된 게시물 분기 포함), 기존 시스템과의 연계 지점, In/Out/Not-doing 세 갈래로 나눈 MVP 컷 라인, 체크 가능한 성공 기준, 그리고 확인이 필요한 가정 사항까지 스킬의 템플릿 구조를 그대로 따랐다. 북마크 분류/태그, 알림, 공유 같은 기능은 "이후"로, 익명 사용자 북마크나 export/import는 "하지 않음"으로 명확히 분리해 스코프가 좁혀지도록 했다.
